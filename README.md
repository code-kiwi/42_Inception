## Objective
The goal of this project is to design and deploy a containerized web infrastructure
using Docker and Docker Compose, following strict configuration and security constraints.

## Architecture
The infrastructure is composed of multiple isolated services:

- NGINX (reverse proxy)
- WordPress
- MariaDB

Each service runs in its own container and communicates through a Docker network.

## Key concepts
- Docker image creation
- Docker Compose orchestration
- Service isolation
- Environment configuration
- Basic networking and volumes management

## Technical stack
- Docker
- Docker Compose
- NGINX
- WordPress
- MariaDB
- Linux
